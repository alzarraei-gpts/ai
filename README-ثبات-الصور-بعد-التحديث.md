# تعديل ثبات الصور بعد تحديث الواجهة

استبدل الملف التالي فقط داخل مشروعك:

```text
js/data-service.js
```

## ما الذي يعالجه؟

كانت صفحات الزائر تقرأ أحيانًا بيانات قديمة محفوظة في `localStorage` قبل قراءة ملفات `data/research-dynamic-data.js` أو `data/research-dynamic-data.json`، لذلك قد تبدو الصور كأنها اختفت أو عادت لمسارات قديمة بعد التحديث.

بعد هذا التعديل:

- صفحات الزائر تقرأ من ملفات المشروع فقط.
- لوحة التحكم فقط هي التي تستخدم الحفظ المؤقت داخل المتصفح.
- بعد تصدير JSON/JS واستبدالهما في مجلد `data`، ستقرأ الواجهة الملفات الجديدة مباشرة.

## طريقة العمل الصحيحة بعد التعديل

1. عدّل من `admin.html`.
2. اضغط حفظ للباقة أو التقسيم أو البوت.
3. اضغط `تصدير JSON` واستبدل:
   `data/research-dynamic-data.json`
4. اضغط `تصدير JS` واستبدل:
   `data/research-dynamic-data.js`
5. تأكد أن الصور نفسها موجودة داخل:
   `images/packages` أو `images/sections` أو `images/bots`
6. افتح الواجهة واضغط `Ctrl + F5`.

## ملاحظة مهمة

إذا كان المسار في البيانات:

```text
images/bots/pkg01-sec02-bot01.png
```

فيجب أن يكون الملف موجودًا بهذا الاسم نفسه داخل:

```text
images/bots/
```

إذا كان الملف الموجود اسمه مثلًا:

```text
pkg01-sec02-bot07.png
```

فلن تظهر صورة المسار `bot01` حتى يتم تعديل المسار أو تغيير اسم الملف.
