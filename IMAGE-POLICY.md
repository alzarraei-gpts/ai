# سياسة الصور — أسماء أصلية بدون فرض أسماء ثابتة

تم تعديل لوحة التحكم بحيث لا تفرض أسماء مثل `g00.webp` أو `b001.webp`.

## القاعدة المعتمدة

- صورة الباقة توضع داخل: `images/packages/`
- صورة التقسيم توضع داخل: `images/sections/`
- صورة البوت توضع داخل: `images/bots/`

يمكن إبقاء اسم الصورة كما هو، ثم حفظ المسار النسبي داخل لوحة التحكم.

مثال:

```text
images/sections/اقتراح-عنوان-البحث.webp
images/bots/بوت-عنوان-نوعي.webp
images/packages/research-cover.webp
```

## طريقة العمل

1. ضع الصورة داخل المجلد المناسب.
2. افتح `admin.html`.
3. استورد `data/research-dynamic-data.json` عند الحاجة.
4. في خانة الصورة اكتب المسار النسبي كما هو.
5. اضغط: اعتماد المسار الحالي.
6. اضغط: حفظ الباقة أو حفظ التقسيم أو حفظ البوت.
7. اضغط: تصدير JSON.
8. استبدل ملف `data/research-dynamic-data.json` بالملف الجديد.

## زر اختيار صورة باسمها الأصلي

عند اختيار صورة من الجهاز، تضع لوحة التحكم المسار تلقائيًا بهذا الشكل:

```text
images/sections/اسم-الصورة-الأصلي.webp
```

لكن المتصفح لا يستطيع نقل الصورة فعليًا إلى مجلد المشروع. لذلك يجب أن تنسخ الصورة يدويًا إلى المجلد نفسه حتى تظهر بعد إعادة فتح الصفحة أو بعد النشر.

## ملاحظة للنشر

يفضل استخدام أسماء قصيرة بلا رموز معقدة. الأسماء العربية مقبولة غالبًا، لكن الأسماء الإنجليزية القصيرة أكثر استقرارًا عند النشر على GitHub Pages.
